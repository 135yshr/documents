Claude Code automatically loads a markdown file called CLAUDE.md into context at the start of every session  ￼

③ コンテキスト構造の内訳（かなり使える）

    •    無視できるレベルではない

記事への追加（かなり説得力UP）

Claude Code のコンテキストには以下が含まれます：

- System prompt: 約5,000〜15,000トークン
- CLAUDE.md: 約1,000〜10,000トークン

つまり、CLAUDE.md のサイズは無視できない影響を持ちます。

④ コンテキストは自動圧縮される（重要）

older content is automatically summarized  ￼

⸻

意味

👉 コンテキストは固定じゃない
👉 圧縮される＝「単純なトークン削減論は不完全」

⸻

記事に入れるべき補強

なお、コンテキストが一定量を超えると古い情報は自動的に要約されます。
そのため、単純なトークン削減がそのまま精度向上につながるとは限りません。

👉 これ入れると一気に“浅い記事感”が消える

⸻

⑤ 「必要な情報だけ入れるべき」の根拠

これも重要👇

If you stuff too much information here, the room available for code reading decreases  ￼

⸻

意味

👉 情報が多すぎると逆効果

⸻

記事補強

情報を過剰に詰め込むと、コード理解に使えるコンテキストが減少し、
結果として精度が低下する可能性があります。

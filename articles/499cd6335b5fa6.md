## pull_request_template.md の作り方

GitHubでPRを作るとき、毎回同じ説明を書いていませんか。

PRテンプレートを使えば、この問題は簡単に解決できます。

## PRテンプレートとは

`.github/pull_request_template.md` に配置するMarkdownファイルで、PR作成時に自動表示されます。

## 作り方（3分で終わる）

### 1. ディレクトリを作成

リポジトリのルートで以下を実行します。

```bash
mkdir .github
```

### 2. ファイル作成

```bash
touch .github/pull_request_template.md
```

最終構成は以下のとおりです。

```text
repo-root/
├── .github/
│   └── pull_request_template.md
```

## テンプレート例

```md
## 概要

（何を変更したか）

## 確認方法

（動作確認手順）

## 影響範囲

（関連する機能）

## AI Session

- Entire Checkpoint: （コミットメッセージの `Entire-Checkpoint:` トレーラーを転記）
- Session URL: https://entire.io/<org>/<repo>/checkpoints/<id>
```

commit & pushすれば、次回PR作成時から自動表示されます。GitHub側の設定は不要です。

テンプレートがあると、チーム全員が同じ項目を埋めるため記載内容が標準化されます。レビュアーも「概要→確認方法→影響範囲」の順に読めばよいので、確認にかかる時間が短くなります。
